Build a complete, deployable full-stack web application called GramRoute in Replit – an AI-Powered Mandi Route Price Optimizer for small farmers in rural India. The app helps farmers maximize profits by recommending optimal mandi routes considering transport costs, spoilage, market fees, and real-time prices. Base it exactly on this hackathon project spec: [Paste the full Team_GenZion.pdf summary here or upload the file to Replit].

Core Functionality (build incrementally, checkpoint after each major step):
1. Simple mobile-first frontend: HTML/CSS/JS with Tailwind CSS. Inputs: starting location (text or geolocation), crop type (dropdown: tomato, onion, potato, etc.), quantity (quintals), max travel distance (km), fuel cost per km. Output: Top 3 recommended routes with maps, net profit breakdown, natural language explanations (e.g., "Go to Mandi A first: saves 30% spoilage, +Rs 2000 profit").

2. Backend: Python FastAPI server on port 8000. Endpoint POST /recommend-route that takes inputs, processes, returns JSON with ranked routes.

3. Multi-Agent AI System using LangGraph/LangChain (use free Ollama or Groq if available in Replit; fallback to mock LLM):
   - Data Agent: Load sample mandis.json (create with 50+ Indian mandis: name, lat/lng, crop prices, fees). Filter by radius using Haversine distance.
   - Planner Agent: Generate single-stop and two-stop routes within radius.
   - Cost/Spoilage Agent: For each route, calculate: distance (Haversine), fuel cost, loading/unloading (fixed Rs 200), market fees (1-8.5%), spoilage (crop-specific rate * distance/time, e.g., tomato 2%/100km).
   - Explainer Agent: LLM generates farmer-friendly explanations and tips.

4. Sample Data: Create mandis.json with realistic Tamil Nadu/Maharashtra data (e.g., Koyambedu Chennai: tomato Rs 40/kg, lat 13.08, lng 80.27).

5. Deployment: Auto-run on Replit (frontend proxies backend via fetch). Make responsive, low-bandwidth friendly. Add PWA support for offline.

Tech Stack (install via Replit packages/requirements.txt):
- Frontend: HTML/JS/Tailwind (CDN)
- Backend: FastAPI, Pydantic, Uvicorn, LangChain/LangGraph, scikit-learn (for distances), json
- No external DB initially – use Replit DB or JSON files
- Maps: Leaflet.js with OpenStreetMap (free)
